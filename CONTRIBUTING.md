# MedEcho Internal Development Guide

Welcome to the MedEcho engineering team! This repository contains the proprietary source code for the MedEcho platform.

## Confidentiality
**This is closed-source, proprietary software.** Do not share, fork, or distribute this code outside of the organization.

## Development Workflow

1. **Branching Strategy:**
   - Always create a new branch from `main` for your feature or bug fix.
   - Use the format: `feature/your-feature-name` or `bugfix/issue-description`.

2. **Local Setup:**
   - Ensure you have Python 3.13+ installed.
   - Install dependencies: `pip install -r requirements.txt`.

3. **Code Reviews:**
   - Push your branch to the internal remote.
   - Open a Pull Request on GitHub.
   - Require at least one approval from a lead engineer before merging into `main`.

Thank you for building the future of clinical documentation!
